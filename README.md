#	What is POMOTODO ?
#### POMOTODO는 일정 관리에 특화된 여러기능을 한데 묶은 웹애플리케이션입니다

#### POMOTODO의 기능
 ###### 1. 구글 검색
 ###### 2. 포모도로 타이머
 ###### 3. 투두 리스트
 ###### 4. 낫투두 리스트
 ###### 5. 과거 기록 시각화
 ###### 6. 집중에 도움이 되는 생활 소음
 ###### 7. 크롬 확장 프로그램

## 개발이유  
- 현재 나에게 가장 필요한 서비스??  
  - 공부를 좀 더 효율적으로 하기 위해 매시간 체크하는 pomodoro타이머,
  - 명확한 목표를 설정하기 위한 TodoList
  - 나의 안좋은 습관을 체크하고 고치기 위한 Not TodoList
  - 집중하는데 도움이 되는 생활 소음
  - 작은 보상을 얻기 위한 과거 기록 체크

- 시장에 있는 비슷한 프로그램들을 사용해보고 불편하거나 불만이었던 점을 수정하자
  - 맘에들지 않는 디자인 : 이건 개인적이지만 디자인이 맘에들면 기능이 부족하고 기능이 맘에들면 디자인이 맘에 안들어서 손이 잘 안가는 문제가 있었음
  - 부족한 기능 & 불편한 사용성 : 포모도로시간을 자동으로 기입해서 그 시간에 어떤걸 했는지 한눈에 보고싶었음, 투두리스트, 낫투두리스트작성, 수정, 삭제들이 가능한 한번에 처리가 됬으면 했음, 마우스를 사용하지 않고 키보드만으로 비교적 수월하게 동작하는 프로그램이면 더 사용하기 좋을것같았음
  - 지원하지 않는 기능으로 인해 발생하는 문제점 : 여러개의 창을 띄우고 공부해야하는데 책으로만 공부하면 문제가 비교적 덜하지만 사소한 문제들로 인해 집중이 흐트러지는 경험을 했음
     1. 간편한 포모도로 타이머를 사용하고 다른 메모어플에 시간과 간단한 텍스트를 옮겨적어야함
     2. 투두리스트끼리 연관성을 부여해서 한곳에 시각적으로 모으고 싶은데 입력후에 그룹화 할 수 있는 여건이 부족했음
     3. 집중하는데 청각적인 도움을 받기위해 유튜브를 사용하면 광고시청을 하거나 영상 시간이 다 되서 공부흐름도 같이 끊어지는 경우가 있었음

이것들을 한데 묶어 관리 및 사용할 수 있는 웹애플리케이션을 만들고 싶었음


개발목표와 합쳐서 쓸까?


- 이런 일정 관리 방법들을 실물로 사용하는 것보다 확연히 좋은점이 있어야한다
  - 공책보다 웹애플리케이션이 좋은이유 : 작성, 수정, 삭제가 자유롭고 자동화를 통해서 시간이 절약되며 집중해야할 곳에 집중하기 좋은 여건을 만들어줌
  - 휴대폰 어플리케이션보다 웹어플리케이션이 좋은 이유 : 휴대폰으로 공부하는사람보다 노트북이나 데스크탑으로 공부하는 사람이 월등히 많고 디바이스를 통일해서 공부에 필요한 준비를 획기저으로 줄일 수 있음
  - 포모도로 타이머 : 포모도로 타이머를 사용하다가 가장 힘이빠지고 흐름이 끊어질 때는 깜빡하고 뭔갈 하지않았을 때다, 자동화를 통해 인간이기때문에 할 수 있는 실수를 줄이고 실수하더라도 만회할 수 있는 기회를 부여받을 수 있다
  - 투두리스트 / 낫투두리스트 : 투두리스트 / 낫투두리스트는 시각화나 그룹화 그리고 수정이 굉장히 빈번하게 발생하기 때문에 노트에 직접 작성하는것보다 프로그램의 도움을 받는편이 훨씬 좋다
  - 과거 기록 : 과거기록을 통해 내가 얼마나 열심히했는지 얼마나 놀았는지를 객관적으로 볼 수 있고 결과에 따라 마음을 다잡을 수도 아니면 뿌듯함이나 자신감, 또는 지속할 수 있는 에너지 등등 작은 보상들을 얻을 수 있다
  - 오디오 프로그램 : 외부프로그램을 사용하지 않고 하나로 통일했기 때문에 준비시간을 줄일 수 있고 흐름이 끊길확률도 적어진다
  - 확장프로그램 : 만약 크롬에서 새 탭을 열때마다 일정관리프로그램이 뜬다면? 나를 객관적으로 바라 볼 수 있다
  - 확장프로그램을 사용함으로 인해 발생하는 문제점을 해결하기위해?
    사용자들이 새탭을 생성하는 이유는 검색을 하기 위함이 가장크다
    그런데 POMOTODO가 뜬다면 검색을 위해 address bar를 직접 클릭하거나 즐겨찾기를 눌러서 페이지를 옮기는 등 사소하지만 굉장히 귀찮은 작업을 해야하고 이로인해 사용자의 의도를 방해하는 불편함을 주게된다
    사용자의 입장에서는 키보드에있던 오른손을 마우스로 옮겼다가 타이핑을 하기위해 다시 키보드로 가져오는일은 아주아주아주 귀찮은 일이다
    그래서 구글검색창을 구현했고 만약 adress bar를 사용하고 싶다면 shift + tab을 누르면 바로 사용할 수 있도록 구현했다
    검색창이 POMOTODO를 사용하는데는 방해가 되므로 검색창 외부를 클릭하면 사라지도록 구현했고 shift+enter라는 단축키르 만들어서 마우스를 사용할 필요가 없게 했다
    
  - 

매일 직접 사용하고 있는 프로그램들이기에 사용자의 시선에서 좀 더 냉정하게 평가할 수 있고,  
공부했던 내역이 남는다면 많은 도움이 될 것이라고 판단했음.
